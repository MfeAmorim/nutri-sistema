Antes de Escrever Código — Prepare o Ambiente

Todo programador tem um ambiente de trabalho configurado. O seu vai ser o VS Code com uma extensão chamada Live Server. Sem isso, você escreveria código mas teria que ficar apertando F5 no navegador toda hora para ver o resultado.

💡
O que é o Live Server? Extensão que abre seu HTML no browser e atualiza automaticamente toda vez que você salva. Digita, salva, vê — em tempo real.

📦 Passo 1 — Instalar o Live Server no VS Code

  • Abra o VS Code
  • Na barra lateral esquerda clica no ícone de 4 quadradinhos (Extensions)
  • Na caixa de busca digita: Live Server
  • Instala o de Ritwick Dey — mais de 40 milhões de instalações

📁 Passo 2 — Criar sua pasta de trabalho

  • Cria uma pasta em qualquer lugar do computador. Chama de meu-site
  • No VS Code: File → Open Folder → seleciona a pasta meu-site

📄 Passo 3 — Criar seu primeiro arquivo

  • No painel esquerdo, passa o mouse na pasta e clica no ícone de folha com +
  • Digita o nome: index.html e aperta Enter
⚠️
Por que index.html? Convenção universal — quando você acessa www.site.com sem especificar arquivo, o browser procura automaticamente o index.html. É sempre o ponto de entrada.

🚀 Passo 4 — Abrir com o Live Server

  • Com o index.html aberto, clica com botão direito em qualquer lugar do editor
  • Clica em Open with Live Server
  • O browser abre com o endereço 127.0.0.1:5500 — página em branco por enquanto, normal
💡
O que é 127.0.0.1? Endereço do seu próprio computador. O Live Server cria um servidor local — como se o seu PC fosse temporariamente um servidor web. Só você acessa.

Escreva Seu Primeiro HTML

Agora você vai digitar — não copiar — o código abaixo no arquivo index.html. Digitar é importante: seu cérebro aprende o padrão das tags na prática, não na teoria.

⚠️
Regra desta aula: Não use o botão Copiar agora. Digite cada linha. Se errar, corrija. Errar faz parte do aprendizado.

✍️ O que você vai digitar agora

Com o index.html aberto e o Live Server rodando, delete tudo que tiver no arquivo e digite:

HTML — Digite isso no VS Code
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <title>Minha Primeira Página</title>
</head>
<body>

  <h1>Olá, mundo!</h1>
  <p>Essa é minha primeira página HTML.</p>

</body>
</html>

Depois de digitar, aperta Ctrl+S. O browser atualiza sozinho e mostra o resultado.

🔍 Entendendo cada linha

LinhaO que faz
<!DOCTYPE html>Avisa o browser: isso é HTML moderno
<html lang="pt-BR">Começa o documento, idioma português
<head>Configurações — não aparece na tela
<meta charset="UTF-8">Permite acentos, ç, ã e emojis
<title>Nome que aparece na aba do browser
<body>Tudo que aparece na tela fica aqui
<h1>Título grande — heading nível 1
<p>Parágrafo de texto
🔦
Regra de ouro: toda tag que abre <x> precisa fechar </x>. É como parênteses na matemática — o que abre tem que fechar.

🎯 Exercício — Modifique o código

Com o Live Server rodando, faça essas 4 mudanças e veja atualizar no browser após cada Ctrl+S:

  • Troca o texto do <h1> para o seu nome
  • Muda o <title> para Nutri Amorim — veja a aba do browser mudar
  • Adiciona um segundo <p> com qualquer texto depois do primeiro
  • Adiciona um <h2> entre o h1 e o p — perceba que h2 é menor que h1
💡
Conectando com o Nutri Amorim: o arquivo central-estudos.html começa exatamente assim — com !DOCTYPE, html, head e body. A estrutura é idêntica. O que muda é só o conteúdo dentro.

Os 3 Pilares de Qualquer Site

Pensa assim: um site é como uma pessoa. Cada tecnologia cuida de uma parte:

ParteTecnologiaO que faz
🦴 EsqueletoHTMLDefine o que existe na página
👗 AparênciaCSSDefine como cada coisa parece
🧠 ComportamentoJavaScriptDefine o que acontece ao interagir
💡
Sem HTML, não existe nada. Sem CSS, existe mas é feio. Sem JavaScript, existe e é bonito mas não faz nada. Os três são obrigatórios.

🦴 HTML — O Esqueleto

HTML é feito de tags — etiquetas que dizem ao navegador o que é cada coisa. Uma tag tem abertura e fechamento:

HTML
<div>conteúdo aqui</div>

O <div> abre, o </div> fecha. Tudo dentro pertence a essa "caixinha". No Nutri Amorim, veja a barra do topo:

HTML — Topbar do Nutri Amorim
<div class="topbar">
  <a href="dashboard.html" class="back-btn">← Voltar</a>
  <div class="topbar-title">Central de Estudos</div>
  <div class="badge">5 ferramentas integradas</div>
</div>
  • <div class="topbar"> → cria a caixinha da barra verde escura do topo
  • <a href="..."> → cria um link clicável (o botão "← Voltar")
  • As caixinhas ficam dentro umas das outras — isso se chama aninhamento

👗 CSS — A Aparência

CSS diz: "pegue essa caixinha e pinte ela assim". A sintaxe é sempre:

CSS — Estrutura básica
.nome-da-caixinha {
  propriedade: valor;
}

O CSS da topbar no Nutri Amorim:

CSS — Topbar do Nutri Amorim
.topbar {
  background: var(--escuro);  /* fundo verde escuro */
  padding: 0 32px;         /* espaço interno nas laterais */
  height: 56px;             /* altura fixa */
  display: flex;            /* organiza filhos lado a lado */
  align-items: center;     /* centraliza verticalmente */
  position: sticky;        /* fica grudada no topo ao rolar */
}

🧠 JavaScript — O Comportamento

JavaScript entra quando algo precisa acontecer. Quando você clica numa aba do Nutri Amorim, este código roda:

JavaScript — Troca de abas
function showPanel(id, el) {
  // 1. Esconde TODOS os painéis
  document.querySelectorAll('.panel').forEach(p =>
    p.classList.remove('active')
  );
  // 2. Remove destaque de TODAS as abas
  document.querySelectorAll('.tab-btn').forEach(b =>
    b.classList.remove('active')
  );
  // 3. Mostra SÓ o painel clicado
  document.getElementById('panel-' + id).classList.add('active');
  // 4. Destaca a aba clicada
  if (el) el.classList.add('active');
}
🔦
Analogia do interruptor: Primeiro apaga todas as luzes, depois acende só a que você pediu. É exatamente o que esse código faz com os painéis.

🔗 Como os três trabalham juntos

A aba "Flashcards" do Nutri Amorim funciona com os três ao mesmo tempo:

HTML — Botão da aba
<div class="tab-btn active" onclick="showPanel('flashcards', this)">
  🃏 Flashcards
</div>
CSS — Estilo da aba ativa
.tab-btn.active {
  color: var(--verde);
  border-bottom-color: var(--verde);
}

A Anatomia de um Arquivo HTML

O arquivo do Nutri Amorim tem uma estrutura específica. Cada bloco tem um propósito:

HTML — Esqueleto completo
<!DOCTYPE html>         <!-- diz ao browser: isso é HTML5 -->
<html lang="pt-BR">       <!-- raiz de tudo, idioma PT -->

  <head>                 <!-- cabeça: configurações (não aparece) -->
    <meta charset="UTF-8">  <!-- aceita acentos e emojis -->
    <title>Nutri Amorim</title>  <!-- nome na aba do browser -->
    <style>
      /* TODO o CSS fica aqui */
    </style>
  </head>

  <body>                 <!-- corpo: tudo que aparece na tela -->
    <!-- HTML da página aqui -->

    <script>
      // TODO o JavaScript fica aqui
    </script>
  </body>

</html>

📦 O <head> — Configurações invisíveis

O <head> não aparece na tela. Ele configura o documento. No Nutri Amorim:

  • charset UTF-8 → garante que ç, ã, é e emojis apareçam certo
  • viewport → faz o site se adaptar ao celular (responsividade)
  • link Google Fonts → carrega as fontes Playfair Display e DM Sans
  • <style> → todo o CSS do site fica aqui dentro
HTML — Head do Nutri Amorim
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Central de Estudos — Nutri Amorim</title>
  <link href="https://fonts.googleapis.com/css2?family=Playfair..." rel="stylesheet">
  <style>
    /* aqui vem todo o CSS */
  </style>
</head>

🖥️ O <body> — Tudo que você vê

O <body> é onde vive tudo que aparece na tela. Topbar, abas, cards, flashcards — tudo está dentro do body.

⚙️ Por que o <script> fica no final do <body>?

Repare: o JavaScript sempre fica no final, antes de fechar o </body>. Isso não é acidente — é uma decisão técnica importante:

⚠️
Se o script viesse no <head>, o JavaScript tentaria encontrar os elementos HTML (botões, cards, painéis) antes deles existirem na tela — e daria erro. No final do body, o HTML já foi todo carregado quando o JS roda.

🎨 O que são Variáveis CSS? (var(--verde))

Você viu var(--verde) em vários lugares. Isso é uma variável CSS — uma cor definida uma vez e usada em todo lugar:

CSS — Variáveis do Nutri Amorim
:root {
  --verde: #2d6a4f;         /* verde escuro principal */
  --verde-claro: #52b788;   /* verde médio */
  --verde-neon: #74c69d;    /* verde brilhante */
  --escuro: #1a2e1f;        /* quase preto esverdeado */
  --creme: #f8f4ef;         /* fundo bege claro */
}

/* Uso em qualquer lugar: */
.topbar { background: var(--escuro); }
.btn-verde { background: var(--verde); }
💡
Vantagem: Se quiser mudar a cor verde de todo o site, muda em um único lugar no :root. Sem variáveis, teria que mudar em centenas de lugares.

Depuração — A Arte de Encontrar e Corrigir Erros

Durante o desenvolvimento do Nutri Amorim, dois bugs apareceram. Cada um ensina algo importante sobre como o código funciona.

🐛 Bug 1 — As abas não ficavam ativas ao clicar

O código original era assim:

JavaScript — Código com bug
function showPanel(id) {
  document.querySelectorAll('.panel').forEach(p => p.classList.remove('active'));
  document.querySelectorAll('.tab-btn').forEach(b => b.classList.remove('active'));
  document.getElementById('panel-' + id).classList.add('active');
  event.currentTarget.classList.add('active'); // ← PROBLEMA AQUI
}

O problema: event.currentTarget é uma variável global do navegador — ela existia em versões antigas do Chrome, mas parou de funcionar nos navegadores modernos.

⚠️
Lição: Nunca dependa de variáveis globais implícitas. Sempre passe o que você precisa como parâmetro explícito.

A correção: passamos o próprio elemento clicado como parâmetro this:

HTML + JavaScript — Solução
<!-- HTML: passa "this" (o próprio botão clicado) -->
<div onclick="showPanel('flashcards', this)">Flashcards</div>

// JS: recebe o elemento como parâmetro "el"
function showPanel(id, el) {
  // ... esconde tudo ...
  if (el) el.classList.add('active'); // ✓ confiável
}

🐛 Bug 2 — Unexpected token ';' (2 vezes!)

O JavaScript travava com erro de sintaxe. A causa: um trecho de código foi duplicado por engano dentro dos dados:

JavaScript — Dado corrompido
// ❌ ERRADO — linha duplicada e quebrada
bioestatistica: [
  { tema: 'Prevalência', bioestatistica: [  // ← linha fantasma
  { tema: 'Prevalência', enunciado: '...' },   // ← linha correta

// ✓ CORRETO — linha fantasma removida
bioestatistica: [
  { tema: 'Prevalência', enunciado: '...' },

Aconteceu duas vezes, em seções diferentes. A ferramenta de diagnóstico que usamos foi o Node.js pelo terminal:

Terminal — Como diagnosticar erros de JS
# Extrai o JS do HTML e verifica a sintaxe
node --check arquivo.js

# Saída quando tem erro:
# /tmp/test.js:414
# };
# SyntaxError: Unexpected token ';'

# Saída quando está ok:
# (silêncio = sem erros)
🔍
Lição de depuração: o número da linha no erro (414) aponta onde o JavaScript percebeu o problema — mas o erro real costuma estar algumas linhas antes. Sempre olhe para cima do número indicado.

Espaçamento no Código — As Regras que Ninguém Ensina

Espaçamento no código serve para uma coisa só: facilitar a leitura. O browser ignora completamente quantos espaços você coloca — ele vai renderizar igual. Mas para você e para qualquer outro programador que ler seu código, faz toda a diferença.

💡
O browser não liga para espaços extras. <h1>Oi</h1> e <h1> Oi </h1> produzem o mesmo resultado na tela. Os espaços são para você, não para o browser.

📏 Regra 1 — Indentação: 2 espaços por nível

Cada vez que uma tag fica dentro de outra, você avança 2 espaços. Isso se chama indentação e mostra visualmente quem está dentro de quem.

HTML — Indentação correta
<body>                        <!-- nível 0 — sem espaço -->
  <div class="topbar">          <!-- nível 1 — 2 espaços -->
    <a href="#">Voltar</a>       <!-- nível 2 — 4 espaços -->
    <div class="titulo">        <!-- nível 2 — 4 espaços -->
      <span>Nutri Amorim</span>  <!-- nível 3 — 6 espaços -->
    </div>
  </div>
</body>
🔦
Dica do VS Code: aperte Tab em vez de ficar contando espaços. O VS Code já adiciona 2 espaços automaticamente. E se quiser indentar várias linhas de uma vez, seleciona tudo e aperta Tab.

📏 Regra 2 — Tags curtas ficam na mesma linha

Se o conteúdo da tag é pequeno — uma palavra, um número, um ícone — fica tudo na mesma linha:

HTML — Curto = mesma linha
<!-- ✅ Certo — conteúdo curto, tudo numa linha -->
<h1>Central de Estudos</h1>
<p>Bem-vindo ao Nutri Amorim</p>
<span>🃏</span>

<!-- ❌ Desnecessário — conteúdo curto em 3 linhas -->
<h1>
  Central de Estudos
</h1>

📏 Regra 3 — Tags longas ou com filhos: quebra linha

Se a tag tem vários elementos filhos dentro, cada filho ganha sua própria linha:

HTML — Longo = quebra linha
<!-- ✅ Certo — div com filhos, cada um na sua linha -->
<div class="card">
  <h2>Flashcards</h2>
  <p>Revise seus conteúdos</p>
  <button>Iniciar</button>
</div>

<!-- ❌ Difícil de ler — tudo amontoado -->
<div class="card"><h2>Flashcards</h2><p>Revise</p><button>Iniciar</button></div>

📏 Regra 4 — CSS: uma propriedade por linha

No CSS, cada propriedade fica na sua própria linha. Nunca escreva tudo numa linha só — fica impossível de achar o que você quer editar:

CSS — Espaçamento correto
/* ✅ Certo — uma propriedade por linha */
.topbar {
  background: #1a2e1f;
  height: 56px;
  padding: 0 32px;
  display: flex;
}

/* ❌ Difícil de editar — tudo numa linha */
.topbar { background: #1a2e1f; height: 56px; padding: 0 32px; display: flex; }

📏 Regra 5 — Linha em branco entre blocos

Deixe uma linha em branco entre seções diferentes. Isso divide visualmente o código em partes, igual parágrafo num texto:

HTML — Linhas em branco entre seções
<!-- ✅ Com linha em branco — fácil de navegar -->
<div class="topbar">
  <span>Logo</span>
</div>
                              <!-- linha em branco aqui -->
<div class="conteudo">
  <h1>Bem-vindo</h1>
</div>
                              <!-- linha em branco aqui -->
<div class="rodape">
  <p>Nutri Amorim 2026</p>
</div>

📏 Regra 6 — JavaScript: espaço antes da chave e entre operadores

No JavaScript há dois padrões importantes que todo programador segue:

JavaScript — Espaçamento padrão
// ✅ Certo — espaço antes da chave e entre operadores
function calcular(peso, altura) {
  const imc = peso / (altura * altura);
  if (imc > 25) {
    return "acima do peso";
  }
}

// ❌ Sem espaços — difícil de ler
function calcular(peso,altura){
  const imc=peso/(altura*altura);
  if(imc>25){return "acima do peso";}
}

🤖 O VS Code faz muito disso automaticamente

Você não precisa memorizar tudo isso de uma vez. O VS Code tem um recurso chamado Format Document que arruma o espaçamento do seu arquivo inteiro automaticamente:

  • Windows: Shift + Alt + F
  • Mac: Shift + Option + F
  • Ou clique direito no editor → Format Document
🔦
Hábito profissional: antes de salvar qualquer arquivo, aperte Shift+Alt+F para formatar. Com o tempo isso vira reflexo automático e seu código sempre fica organizado.
